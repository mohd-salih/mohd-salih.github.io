Active Directory Configuration & Access Control

2024
Identity & Access Management
Active Directory Windows Server Group Policy RBAC PowerShell

Project Overview

This project involved designing and implementing a comprehensive Active Directory infrastructure from the ground up, focusing on security best practices, organizational efficiency, and role-based access control. The implementation demonstrated proficiency in identity and access management, Windows Server administration, and enterprise security architecture.

The project encompassed domain controller setup, organizational unit design, user and group management, delegation of administrative rights, and implementation of security policies through Group Policy Objects. Special emphasis was placed on the principle of least privilege and defense-in-depth security strategies.

Key Features Implemented

Domain Controller Setup

Configured Windows Server as Active Directory Domain Controller with DNS integration

Organizational Unit Structure

Designed and implemented hierarchical OU structure for efficient resource management

User & Group Management

Created and managed user accounts, security groups, and distribution lists

Delegation Control

Implemented role-based delegation for secure administrative task distribution

Group Policy Objects

Configured GPOs for security policies, password requirements, and access controls

PowerShell Automation

Developed scripts for bulk user creation and automated administrative tasks

Technical Implementation

1. Active Directory Domain Services Setup

Installed and configured Active Directory Domain Services (AD DS) on Windows Server. Promoted the server to a domain controller and configured DNS services for proper domain resolution. Established the forest and domain structure with appropriate naming conventions and security considerations.

2. Organizational Unit Architecture

Designed a hierarchical OU structure aligned with organizational departments and security requirements:

  • Corporate OU: Top-level OU for corporate resources
  • Departments: IT, HR, Finance, Sales, Operations sub-OUs
  • Security Groups: Role-based security groups within each department
  • Service Accounts: Dedicated OU for service account management
  • Workstations: Computer objects organized by location and type
  • Servers: Production and development server segregation

3. User Account Management

Created and configured user accounts with the following considerations:

  • Implemented naming conventions for consistency
  • Configured user principal names (UPNs) and SAM account names
  • Set password policies including complexity, length, and expiration
  • Configured account lockout policies to prevent brute-force attacks
  • Implemented fine-grained password policies for privileged accounts
  • Set up home directories and user profile paths

4. Security Group Configuration

Established security groups following the AGDLP (Accounts, Global, Domain Local, Permissions) best practice:

  • Created global security groups for user role assignments
  • Configured domain local groups for resource permissions
  • Implemented nested group membership for administrative efficiency
  • Set up distribution lists for email communication

5. Delegation of Control

Implemented role-based administrative delegation to distribute management responsibilities securely:

  • Delegated user account creation rights to HR department
  • Granted password reset permissions to helpdesk team
  • Configured OU-level permissions for department administrators
  • Implemented least privilege principle for all delegations
  • Documented all delegation assignments for audit purposes

6. Group Policy Implementation

Configured comprehensive Group Policy Objects for security and management:

  • Password Policy: Minimum 12 characters, complexity requirements, 90-day expiration
  • Account Lockout: 5 failed attempts, 30-minute lockout duration
  • Security Settings: Disabled LM hash storage, enabled SMB signing
  • Audit Policy: Enabled logging for account logon and object access events
  • Software Restrictions: Implemented AppLocker for application control
  • Drive Mapping: Automated network drive assignments based on group membership

Security Enhancements

Privileged Access Management

  • Separated standard user and administrative accounts
  • Implemented Protected Users security group for high-value accounts
  • Configured Kerberos constrained delegation where necessary
  • Disabled legacy authentication protocols (NTLMv1)

Access Control Policies

  • Implemented time-based access restrictions for sensitive resources
  • Configured logon hour restrictions for after-hours security
  • Set up workstation restrictions for administrative accounts
  • Implemented just-in-time (JIT) access for privileged operations

Audit and Compliance

  • Enabled advanced audit policies for security event tracking
  • Configured event log forwarding for centralized monitoring
  • Implemented regular access reviews and certification processes
  • Created compliance reports for regulatory requirements

PowerShell Automation

Developed PowerShell scripts to automate common administrative tasks and improve operational efficiency. Key automation scripts included:

  • Bulk user account creation from CSV files
  • Automated group membership management
  • Password reset and account unlock utilities
  • Inactive account detection and reporting
  • Security group membership auditing
  • OU structure replication for new departments
