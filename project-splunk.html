2024
Data Analysis & Visualization

Project Overview

This project focused on implementing Splunk Enterprise as a centralized log management and security analytics platform. The implementation demonstrated proficiency in data ingestion, SPL query development, dashboard creation, and real-time security event monitoring. Through this project, I established a comprehensive security monitoring infrastructure capable of detecting, analyzing, and visualizing security threats across the enterprise environment.

The project encompassed the entire lifecycle of a Splunk deployment, from initial setup and data source configuration to advanced correlation searches and custom dashboard development. Special emphasis was placed on detecting authentication anomalies, particularly brute-force login attempts, and creating actionable intelligence through data visualization.

Technical Implementation

1. Splunk Infrastructure Deployment

Installed and configured Splunk Enterprise on a dedicated server environment. Set up universal forwarders on Windows and Linux systems to collect and forward logs to the central Splunk indexer. Configured appropriate indexes and sourcetypes for efficient data organization and retrieval.

2. Data Source Configuration

Configured multiple data inputs including Windows Event Logs, Linux syslogs, authentication logs, web server access logs, and firewall logs. Implemented proper data parsing and field extraction to ensure structured data ingestion and efficient searching capabilities.

3. Dashboard Development

Created comprehensive security dashboards featuring:

  • Authentication activity overview with successful and failed login visualizations
  • Top users and source IPs generating authentication events
  • Geographic visualization of login attempts
  • Timeline charts showing authentication patterns over time
  • Real-time metrics for security event volume and velocity
  • Drill-down capabilities for detailed investigation

4. SPL Query Examples

Brute-Force Detection Query:

index=main sourcetype=WinEventLog:Security EventCode=4625
| stats count by src_ip, user
| where count > 5
| table src_ip, user, count
| sort -count

Successful Login After Failed Attempts:

index=main sourcetype=WinEventLog:Security (EventCode=4625 OR EventCode=4624)
| transaction user maxspan=5m
| search EventCode=4625 EventCode=4624
| table _time, user, src_ip, EventCode

Authentication Activity Over Time:

index=main sourcetype=WinEventLog:Security (EventCode=4624 OR EventCode=4625)
| timechart span=1h count by EventCode

5. Real-time Alerting

Configured real-time correlation searches to detect suspicious patterns including:

  • Multiple failed login attempts from a single source IP
  • Login attempts from unusual geographic locations
  • Authentication activity during off-hours
  • Lateral movement indicators through authentication logs
  • Privilege escalation attempts

Dashboard Components

  • Authentication Overview: Real-time statistics on login success/failure rates and trends
  • Top Failed Logins: Visualization of accounts and IPs with highest failed login attempts
  • Geographic Login Map: World map showing authentication attempt origins
  • Timeline Analysis: Historical authentication patterns with anomaly detection
  • User Activity: Individual user authentication behavior and patterns
  • Alert Summary: Real-time display of triggered security alerts and their severity
  • System Health: Splunk infrastructure monitoring and data ingestion metrics

Security Use Cases

Brute-Force Attack Detection

Implemented advanced correlation logic to identify brute-force authentication attempts by analyzing failed login patterns, velocity of attempts, and source IP reputation. Configured automated alerts to notify security team when thresholds are exceeded.

Anomaly Detection

Utilized Splunk's statistical functions to establish baselines for normal authentication behavior and detect deviations that could indicate compromise, including unusual login times, new geographic locations, and abnormal access patterns.

Threat Hunting

Developed SPL queries for proactive threat hunting activities, searching for indicators of compromise, suspicious command executions, and potential lateral movement within the network based on authentication logs and system events.
