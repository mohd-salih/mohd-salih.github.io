Project Overview

This comprehensive security monitoring project involved deploying and configuring Wazuh SIEM (Security Information and Event Management) to provide centralized security monitoring, threat detection, and incident response capabilities across Windows and Linux endpoints. The implementation focused on establishing a robust security operations infrastructure capable of detecting, analyzing, and responding to security threats in real-time.

The project encompassed end-to-end SIEM deployment, from initial infrastructure setup to advanced integration with threat intelligence platforms and network intrusion detection systems. Through this implementation, I demonstrated proficiency in security architecture, log management, threat detection, and automated response mechanisms.

Technical Implementation

1. Wazuh Infrastructure Setup

Deployed Wazuh manager on a dedicated server environment, configured agent-based monitoring for Windows and Linux endpoints, and established secure communication channels between agents and the central management server. The infrastructure was designed for scalability and high availability.

2. File Integrity Monitoring Configuration

Implemented comprehensive FIM on critical system directories including /etc, /bin, /sbin on Linux systems and System32, Program Files on Windows. Configured real-time monitoring with baseline creation and alert generation for any unauthorized modifications. 

<syscheck>
  <directories check_all="yes">/etc,/usr/bin,/usr/sbin</directories>
  <directories check_all="yes">C:\Windows\System32</directories>
  <frequency>43200</frequency>
  <alert_new_files>yes</alert_new_files>
</syscheck>

3. VirusTotal Integration

Integrated VirusTotal API to automatically analyze file hashes detected during FIM scans. Configured active response to quarantine files identified as malicious by multiple antivirus engines, providing automated threat mitigation.

4. Suricata IDS Integration

Deployed Suricata network intrusion detection system and integrated it with Wazuh for centralized alert management. Configured custom rulesets for detecting network-based attacks, port scans, and suspicious traffic patterns.

5. Brute-Force Attack Detection

Implemented correlation rules to detect authentication brute-force attempts across SSH, RDP, and web applications. Configured threshold-based alerting and automated IP blocking for repeated failed login attempts.

MITRE ATT&CK Mapping

Aligned detection capabilities with the MITRE ATT&CK framework to ensure comprehensive coverage of adversary tactics and techniques:

  • Initial Access: Detection of brute-force attacks (T1110) and exploitation attempts
  • Persistence: FIM monitoring for registry modifications (T1547) and startup item changes
  • Privilege Escalation: Monitoring for suspicious privilege changes and sudo abuse (T1548)
  • Defense Evasion: Detection of log deletion (T1070) and security tool tampering
  • Credential Access: Monitoring for credential dumping (T1003) and password spray attacks
  • Discovery: Detection of network and system reconnaissance activities (T1046, T1082)
  • Command and Control: Network traffic analysis for C2 communication patterns
